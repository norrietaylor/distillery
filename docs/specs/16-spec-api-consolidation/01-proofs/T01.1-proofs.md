# T01.1 Proof Summary: Extend store layer -- stale_days, group_by, output=stats

## Task
Extend the DistilleryStore protocol and DuckDB implementation to support three new
`list_entries` parameters: `stale_days`, `group_by`, and `output`.

## Changes Made

### `src/distillery/store/protocol.py`
- Added keyword-only parameters to `list_entries`: `stale_days: int | None = None`,
  `group_by: str | None = None`, `output: str | None = None`
- Updated return type to `list[Entry] | dict[str, Any]`
- Added comprehensive docstring for new parameters

### `src/distillery/store/duckdb.py`
- Updated `list_entries` signature to match protocol
- **stale_days**: Adds `COALESCE(accessed_at, updated_at) < NOW() - INTERVAL N DAYS`
  WHERE clause, composable with all existing filters
- **group_by**: Delegates to existing `aggregate_entries` method (which already handles
  GROUP BY with CTE for totals). Added `"tags": "UNNEST(tags)"` to `_AGGREGATE_EXPR_MAP`
  so `group_by="tags"` unnests the tags array. Tag prefix filtering works via existing
  `tag_prefix` filter key in `_build_filter_clauses`.
- **output="stats"**: New `_get_entry_stats` private method returns `entries_by_type`,
  `entries_by_status`, `total_entries`, and `storage_bytes` (via `PRAGMA database_size`).
  Respects both `filters` and `stale_days`.
