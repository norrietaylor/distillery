## Evidence

### Test Coverage

**TestBaselineSaveFormat (5 tests)**
- Baseline JSON contains `scenarios` and `cost_summary` keys
- `cost_summary` contains `total_cost_usd`, `total_tokens`, `per_skill`
- Per-scenario fields: `input_tokens`, `output_tokens`, `total_tokens`, `total_cost_usd`
- Per-scenario `name` and `skill` fields
- `cost_summary.total_cost_usd` equals sum of all scenario costs

**TestPerSkillAggregation (4 tests)**
- `per_skill` keys match distinct skills in results
- Per-skill `cost_usd` sums correctly across scenarios
- Per-skill `scenario_count` counts correctly
- Per-skill `tokens` sums `total_tokens` per skill

**TestCostComparison (3 tests)**
- `total_delta_usd` = current minus baseline total
- `per_skill` delta includes `delta_usd`, `baseline_usd`, `current_usd`
- `total_delta_pct` is correctly calculated

**TestCostWarning (4 tests)**
- Warning emitted when skill cost increases >20%
- No warning when cost increases <20%
- No warning when cost decreases
- Boundary condition: exactly +20% does not trigger warning

**TestBackwardCompatibility (2 tests)**
- Old-format flat-list baseline does not crash `--compare-cost`
- Old-format baseline produces `"no cost baseline available"` note

**TestMissingCostSummary (2 tests)**
- New-format baseline dict without `cost_summary` key reports graceful message
- Missing `cost_summary` does not raise an exception
