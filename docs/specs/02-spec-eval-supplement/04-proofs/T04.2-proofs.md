## Implementation Summary

Changes made to `src/distillery/cli.py`:

1. **Argument parser** (line ~134): Added `--compare-cost` flag as `store_true` argument
2. **`_cmd_eval` signature** (line ~466): Added `compare_cost: bool = False` parameter
3. **JSON output** (line ~544): Moved `summary` dict construction outside the `if fmt == "json"` block; deferred JSON printing when `compare_cost=True` so `cost_comparison` key can be appended
4. **Cost comparison logic** (line ~618): After regression check, if `compare_cost=True`:
   - Loads `cost_summary` from baseline (handles missing gracefully)
   - Computes total delta (absolute + percentage)
   - Computes per-skill deltas
   - Flags any skill with >20% increase as WARNING
   - Text: appends "Cost Comparison:" block after pass/fail summary
   - JSON: adds `cost_comparison` key to the output dict
   - Exit code: cost warnings do NOT change exit code (only assertion failures do)
5. **`main()`** (line ~756): Passes `compare_cost=getattr(args, "compare_cost", False)` to `_cmd_eval`

## Key Design Decisions

- `summary` dict built unconditionally (not inside `if fmt == "json"`) so it can be mutated with `cost_comparison` key in cost comparison logic
- JSON printing deferred when `compare_cost=True` to avoid double-printing
- No baseline: prints "no cost baseline available" note (text) or `{"note": "no cost baseline available"}` (JSON)
- Exit code unchanged: only scenario failures trigger exit code 1
