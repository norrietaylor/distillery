## What Was Implemented

### promptfooconfig.yaml (repo root)

A promptfoo configuration file that:

1. **Provider**: `anthropic:messages:claude-sonnet-4-5-latest` with Distillery MCP server
   attached via stdio transport (`distillery-mcp` command)

2. **MCP Integration**: Uses promptfoo's native `config.mcp.servers` to connect to the
   Distillery MCP stdio server. This is promptfoo's native MCP support (no custom
   wrapper needed).

3. **10 Smoke-Test Scenarios** covering all required skill paths:
   - `/distill`: 2 scenarios (store new entry, dedup check)
   - `/recall`: 2 scenarios (search with results, empty store graceful handling)
   - `/bookmark`: 2 scenarios (store URL, dedup check for known URL)
   - `/pour`: 2 scenarios (synthesize architecture decisions, empty store)
   - `/watch`: 2 scenarios (list sources, add RSS source)

4. **Assertions per scenario**:
   - `javascript` assertion: checks expected MCP tool calls are made (tool names)
   - `latency` assertion: enforces wall-clock limits (30s-120s per skill type)
   - `not-regex` assertion: verifies absence of stack traces in responses
   - `icontains` assertion: verifies expected content substrings where applicable

5. **Output**: `.promptfoo/results.json` (covered by .gitignore)

### .gitignore update

Added `.promptfoo/` to `.gitignore` to exclude promptfoo's local output directory
from version control (required by spec Unit 1 functional requirements).
