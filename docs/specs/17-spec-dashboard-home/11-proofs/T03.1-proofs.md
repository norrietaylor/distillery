## Summary

### Implementation

Created two new Svelte components and a test file:

1. **CorrectionCard.svelte** — Expandable card component that shows original vs corrected content side by side when clicked. Uses aria-expanded, role="button", and keyboard (Enter/Space) support.

2. **RecentCorrections.svelte** — Section component that:
   - Fetches session entries from the last 7 days via `list(entry_type="session", date_from=<7d_ago>)`
   - For each entry, checks for `corrects` relations via `relations(action="get", entry_id=<id>, relation_type="corrects")`
   - Fetches original entry content via `get(id=<original_id>)`
   - Scopes queries to `selectedProject` when set
   - Displays loading skeleton, error alert, empty state, or list of CorrectionCards
   - Reactive to `refreshTick` and `selectedProject` store changes

3. **RecentCorrections.test.ts** — 17 unit tests covering all required scenarios from the feature spec (loading, empty state, display, error handling, project scoping, date filtering, expand/collapse).

### App.svelte Change

Added `RecentCorrections` import and `<RecentCorrections {bridge} />` to the home section.
