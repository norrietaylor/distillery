## Implementation Summary

Replaced the stub `_handle_maintenance` function in
`src/distillery/mcp/webhooks.py` with a full implementation that
sequentially executes five KB maintenance operations and stores a
digest entry.

### Changes Made

1. Added `timedelta` to the `datetime` import.
2. Added `_MAINTENANCE_PERIOD = timedelta(days=7)` constant.
3. Added `_parse_mcp_response()` helper that extracts JSON from MCP
   `TextContent` response lists returned by the server handler functions.
4. Replaced the `_handle_maintenance` stub with a full implementation that:
   - Calls `_handle_metrics` (7-day period), `_handle_quality`,
     `_handle_stale` (30 days, limit 10), `_handle_interests` (30 days,
     top 10), and `_handle_suggest_sources` (max 3) sequentially.
   - Parses each MCP response via `_parse_mcp_response()`.
   - Composes a one-paragraph digest summary from the aggregate results.
   - Creates and stores an `Entry` with `entry_type=EntryType.SESSION`,
     `author="distillery-maintenance"`, tags `["system/digest",
     "system/weekly", "system/maintenance"]`, and metadata containing
     `period_start` and `period_end` ISO dates.
   - Returns `{"ok": true, "data": {"metrics": {...}, "quality": {...},
     "stale_count": N, "top_interests": [...], "suggested_sources": [...],
     "digest_entry_id": "<uuid>"}}`.
   - Returns 500 with `{"ok": false, "error": "<message>"}` on any
     unhandled exception (no digest stored on error).
   - Logs at INFO on start and completion.
