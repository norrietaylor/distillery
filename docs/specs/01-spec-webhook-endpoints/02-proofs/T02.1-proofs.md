## Implementation Summary

Replaced stub `_handle_poll` and `_handle_rescore` functions in
`src/distillery/mcp/webhooks.py` with full implementations.

### Changes Made

1. Added `import json` to module imports.
2. Updated `_handle_poll` signature to accept `state: dict[str, Any]` in
   addition to `request`.  The handler instantiates `FeedPoller` with the
   store and config from shared state, calls `await poller.poll()`, collects
   per-source errors from the `PollerSummary`, and returns
   `{"ok": true, "data": {"sources_polled": N, "items_fetched": N,
   "items_stored": N, "errors": [...]}}`.  Returns 500 on exception.
   Logs at INFO on start and completion.
3. Updated `_handle_rescore` signature similarly.  Parses optional JSON body
   for `{"limit": N}` (default 200), returns 400 on malformed body,
   instantiates `FeedPoller`, calls `await poller.rescore(limit=N)`, and
   returns `{"ok": true, "data": {"rescored": N, "upgraded": N,
   "downgraded": N}}`.  Returns 500 on exception.  Logs at INFO on start
   and completion.
4. Updated `_handle_maintenance` stub to accept the new `(request, state)`
   signature for consistency.
5. Updated `_authenticated_endpoint` dispatcher to pass `state` as the
   second argument when calling `handler(request, state)`.
