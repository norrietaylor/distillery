# T01.3 Proof Summary: Wire implicit feedback into MCP server handlers

## Task

Integrate search logging and implicit feedback recording into the MCP server handlers
`_handle_search()` and `_handle_get()`.

## Implementation

### Changes Made

**`src/distillery/mcp/server.py`**

1. Added `datetime` to the import from `datetime` (already had `UTC`).

2. In `_lifespan()`: initialized `_state["recent_searches"] = []` after the store is
   created. Each element is a dict with `search_id` (str), `entry_ids` (set[str]),
   and `timestamp` (datetime).

3. Updated `_handle_search()` signature to accept optional `recent_searches: list | None = None`.
   After returning results, calls `store.log_search()` and appends a record to
   `recent_searches`. Errors are logged but do not interrupt the response.

4. Updated `_handle_get()` signature to accept optional `recent_searches: list | None = None`
   and `config: DistilleryConfig | None = None`. On each call:
   - Reads `feedback_window_minutes` from `config.classification.feedback_window_minutes`
     (defaults to 5 if config is None).
   - Prunes expired entries from `recent_searches` in-place.
   - For any non-expired search record that includes `entry_id`, calls
     `store.log_feedback(search_id, entry_id, "positive")`.

5. Updated `_call_tool()` dispatcher to pass `_state.get("recent_searches", [])` and
   `cfg` to both handlers.

Both new parameters are optional (default `None`) so existing unit tests calling the
handlers directly continue to pass without modification.

**`tests/test_e2e_mcp.py`**

Added `"distillery_check_conflicts"` to the expected tool set in
`test_create_server_registers_all_tools` (this tool was added in T03.2 but the test
had not been updated).
