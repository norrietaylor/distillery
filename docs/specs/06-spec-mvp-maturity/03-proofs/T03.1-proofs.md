## Design Notes

- `ConflictChecker` follows the same prompt/response separation pattern as `ClassificationEngine`: it builds prompts and parses responses but does NOT call the LLM directly.
- `check()` with `llm_responses=None` queries the store but returns an empty `ConflictResult`, enabling a two-pass workflow where the caller collects LLM responses before a second `check()` call.
- `parse_response()` handles parse failures gracefully, returning `(False, "")` and logging a warning, consistent with `ClassificationEngine.parse_response()`.
- Default threshold of `0.60` matches the task spec requirement.
