## Key Design Decisions

1. **Graceful table absence**: `search_log` and `feedback_log` tables (added by T01.1/T02.1) may not exist yet. The implementation queries `information_schema.tables` first and returns zeros if absent, catching any exceptions too.

2. **accessed_at fallback**: The `accessed_at` column (added by T02.1) may not exist. Implementation tries `COALESCE(accessed_at, updated_at)` and falls back to `updated_at` only on exception.

3. **period_days dynamic keys**: Activity and search sections include `created_{period_days}d` / `searches_{period_days}d` keys in addition to fixed 7/30/90 day windows.

4. **Read-only**: No data is modified. Verified by test assertion.
