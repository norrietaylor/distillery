## Verification Details

### _track_version_info behavior
1. Reads `schema_version` from `_meta` (defaults to `"0"` on fresh database)
2. Reads `duckdb.__version__` for current DuckDB version
3. Compares stored `duckdb_version` with current; logs WARNING if major or minor differs
4. Upserts `duckdb_version` and `vss_version` into `_meta` using `ON CONFLICT DO UPDATE`
5. Ensures `schema_version` key exists in `_meta` (inserts `"0"` if missing via `ON CONFLICT DO NOTHING`)
6. Logs: `"Schema at version {N}, DuckDB {version}"`

### Tests added
- `test_meta_version_tracking` — verifies `schema_version` and `duckdb_version` written to `_meta`
- `test_meta_duckdb_version_updated_on_reopen` — verifies `duckdb_version` is updated to current on re-init
- `test_meta_schema_version_default_zero` — verifies fresh database defaults `schema_version` to `"0"`
