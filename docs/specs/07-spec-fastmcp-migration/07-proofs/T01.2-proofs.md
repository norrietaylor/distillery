# T01.2 Proof Summary: FastMCP Server Scaffold with Lifespan Context Manager

## Task
Refactor `create_server()` in `src/distillery/mcp/server.py` to construct a `FastMCP("distillery", lifespan=lifespan)` instance instead of `Server("distillery", lifespan=_lifespan)`. Implement the lifespan as an async context manager that yields a dict containing `store`, `config`, `embedding_provider`, and `recent_searches`. Update imports from `mcp` to `fastmcp`.

## Changes Made

### `src/distillery/mcp/server.py`
- Replaced `from mcp.server import Server` and `from mcp.server.stdio import stdio_server` with `from fastmcp import Context, FastMCP`
- Kept `from mcp import types` for `types.TextContent` used in `error_response()` and `success_response()` (unchanged per spec)
- Changed `create_server()` return type from `Server` to `FastMCP`
- Changed `Server("distillery", lifespan=_lifespan)` to `FastMCP("distillery", lifespan=lifespan)`
- Refactored lifespan from using mutable `_state` dict to yielding a dict: `{"store": store, "config": config, "embedding_provider": embedding_provider, "recent_searches": recent_searches}`
- Removed `@server.list_tools()` and `@server.call_tool()` decorators (these use the old mcp.Server API and will be replaced in T02 with `@mcp.tool()` decorators)
- Updated `run_server()` to use `server.run_stdio_async(show_banner=False)` instead of manual `stdio_server()` context manager

### `tests/test_mcp_server.py`
- Updated `TestCreateServer::test_create_server_returns_server_instance` to check `isinstance(server, FastMCP)` instead of `isinstance(server, Server)`
- Updated `TestCreateServer::test_server_registers_all_fifteen_tools` to use `await server.list_tools()` (FastMCP API) instead of `server.request_handlers[mcp_types.ListToolsRequest]`
