### test_store_integration.py (25 tests)

**TestStoreGetRoundTrip (5 tests)**
- Full round-trip: store -> get preserves content, metadata, tags, type
- Multiple entries stored independently without corruption

**TestStoreSearchFlow (6 tests)**
- Stored entry is retrievable via semantic search
- SearchResult objects have float scores
- Similar content ranks higher than dissimilar content (controlled vectors)
- entry_type and author filters applied correctly
- limit parameter respected

**TestFindSimilarIntegration (6 tests)**
- Identical content (cosine similarity ~1.0) identified as duplicate
- High threshold filters out dissimilar entries (orthogonal vectors)
- All returned scores are >= threshold
- Results ordered by descending similarity score
- Empty store returns empty list
- limit parameter respected

**TestUpdateEmbeddingRefresh (3 tests)**
- update(content=...) refreshes the embedding vector in the DB
- get() after update() reflects new content
- version counter increments on each update

**TestDeleteIntegration (3 tests)**
- Soft-deleted entries remain in DB with archived status
- Active entries appear in list_entries without filters
- delete() returns False for non-existent entries

**TestMetaTableIntegration (2 tests)**
- _meta table records embedding_model and embedding_dimensions on first init
- Re-opening DB with different model raises RuntimeError (mismatch protection)
