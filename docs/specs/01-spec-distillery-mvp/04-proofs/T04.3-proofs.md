## What Was Implemented

Three new MCP tools added to `/Users/norrie/code/distillery/src/distillery/mcp/server.py`:

### distillery_search
- Required: `query` (string)
- Optional: `limit` (int, default 10, range 1-200), plus filter fields:
  `entry_type`, `author`, `project`, `tags`, `status`, `date_from`, `date_to`
- Returns: `{"results": [{"score": float, "entry": {...}}, ...], "count": N}`
- Delegates to `store.search(query, filters, limit)`

### distillery_find_similar
- Required: `content` (string)
- Optional: `threshold` (float, default 0.8, range [0.0, 1.0]), `limit` (int, default 10, range 1-200)
- Returns: `{"results": [{"score": float, "entry": {...}}, ...], "count": N, "threshold": T}`
- Delegates to `store.find_similar(content, threshold, limit)`
- Designed for deduplication before storing new content

### distillery_list
- All fields optional
- Optional: `limit` (int, default 20, range 1-500), `offset` (int, default 0, >= 0)
- Filter fields: same as search (entry_type, author, project, tags, status, date_from, date_to)
- Returns: `{"entries": [...], "count": N, "limit": L, "offset": O}`
- Results ordered newest-first (no semantic ranking)
- Delegates to `store.list_entries(filters, limit, offset)`

### Helper function
- `_build_filters_from_arguments(arguments)`: Extracts filter keys from the tool
  argument dict into a filters dict for the store. Returns `None` if no filters present.
