## Implementation Summary

Implemented three MCP tools in `src/distillery/mcp/server.py`:

- `distillery_store` -- Accepts `content`, `entry_type`, `author`, optional `project`, `tags`, `metadata`, `dedup_threshold` (default 0.92), `dedup_limit` (default 3). Creates an `Entry` with `source=claude-code`, persists it via `DuckDBStore.store()`, runs `find_similar` for deduplication warnings, and returns `{"entry_id": ..., "warnings": [...]}`.
- `distillery_get` -- Accepts `entry_id`. Returns the full serialised entry dict or a `NOT_FOUND` structured error.
- `distillery_update` -- Accepts `entry_id` plus any combination of updatable fields (`content`, `entry_type`, `author`, `project`, `tags`, `status`, `metadata`). Validates enum fields, rejects immutable field updates (`id`, `created_at`, `source`), and returns the updated entry or a structured error.

Supporting module-level constants added:
- `_VALID_ENTRY_TYPES` -- set of valid entry type strings
- `_VALID_STATUSES` -- set of valid status strings
- `_IMMUTABLE_FIELDS` -- set of fields that cannot be updated
- `_DEFAULT_DEDUP_THRESHOLD = 0.92`
- `_DEFAULT_DEDUP_LIMIT = 3`

Note: Worker-21 (T04.3) added tool definitions for `distillery_search`, `distillery_find_similar`, and `distillery_list` to the tool registry and dispatch before T04.2 was complete. Those handler stubs were already present in the file. T04.2 adds the real handler functions alongside the T04.3 handlers.
