## Implementation Details

### OpenAIEmbeddingProvider

Located in `src/distillery/embedding/openai.py`. Key design decisions:

1. **Protocol compliance**: Implements `embed(text)`, `embed_batch(texts)`, `dimensions` property, and `model_name` property as required by `EmbeddingProvider` protocol.

2. **API key resolution**: Accepts `api_key` directly or reads from environment variable named by `api_key_env` (default: `OPENAI_API_KEY`). Raises clear `ValueError` if no key is found.

3. **Rate limiting with exponential backoff**: `embed_batch()` uses `_RateLimitError` and `_ServerError` internal exception classes to trigger retries on HTTP 429 and 5xx responses. Backoff is `2^attempt` seconds (1s, 2s, 4s). Raises `RuntimeError` after 3 failed attempts.

4. **Response parsing**: Sorts API response items by `index` field to guarantee output order matches input order, even if the API reorders them.

5. **httpx usage**: Uses a persistent `httpx.Client` with 30-second timeout. Closed in `__del__` to avoid resource leaks.

6. **OpenAI dimensions parameter**: Passes `dimensions` in the API payload to request truncated embeddings (Matryoshka-style), matching the OpenAI API's native support for this feature.

### Factory Function `create_provider()`

Located in `src/distillery/embedding/__init__.py`. Reads `config.embedding.provider` and returns the appropriate provider:
- `"openai"` â†’ `OpenAIEmbeddingProvider`
- `"jina"` â†’ `JinaEmbeddingProvider`
- Any other value â†’ raises `ValueError` with a clear message listing supported providers
