## Implementation Summary

### Methods Implemented
- **`search(query, filters, limit)`**: Embeds query via embedding provider, uses `array_cosine_similarity` with HNSW index, applies metadata filters as SQL WHERE clauses, returns `list[SearchResult]` sorted by descending similarity.
- **`find_similar(content, threshold, limit)`**: Embeds content, filters by cosine similarity >= threshold, returns `list[SearchResult]` sorted by descending similarity.
- **`list_entries(filters, limit, offset)`**: Lists entries with metadata filters, pagination via LIMIT/OFFSET, ordered by `created_at` DESC. No semantic ranking.

### Helper Methods Added
- **`_build_filter_clauses(filters)`**: Static method translating filter dict to SQL WHERE fragments with parameterized queries. Supports: `entry_type` (str or list), `author`, `project`, `tags` (via `list_has_any`), `status`, `date_from`, `date_to`.
- **`_row_to_entry(row, columns)`**: Converts DuckDB result row to `Entry` instance, handling JSON metadata parsing and NULL tag arrays.
