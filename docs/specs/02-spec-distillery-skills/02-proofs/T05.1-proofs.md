## Implementation

Created: `.claude/skills/minutes/SKILL.md`

The SKILL.md implements three modes:

**New Meeting Mode** (default):
- Gathers title, attendees, discussion points, decisions, action items, and follow-ups
- Generates `meeting_id` from slugified title + date (e.g., `arch-review-2026-03-22`)
- Formats structured notes with sections for Discussion, Decisions, Action Items, Follow-ups
- Shows preview before storing
- Calls `distillery_store` with `entry_type="minutes"` and metadata `{meeting_id, attendees, version: 1}`
- Confirms with entry ID, meeting_id, version, and content preview

**Update Mode** (`/minutes --update <meeting_id>`):
- Searches for existing entry by `meeting_id` using `distillery_search`
- If not found: reports missing and offers to create new
- If found: gathers additional notes and appends under `## Update — <timestamp>` heading
- Calls `distillery_update` with full updated content and incremented version
- Confirms with entry ID, meeting_id, new version number, and appended section preview

**List Mode** (`/minutes --list`):
- Calls `distillery_list` with `entry_type="minutes"`, `limit=10`
- Displays compact table with meeting_id, title, date, attendee count
