## Summary

Added `promote_entities(threshold, reserved_prefixes=None)` to the
`DistilleryStore` Protocol (`src/distillery/store/protocol.py`) and implemented
it in `src/distillery/store/duckdb.py`. The job:

- scans active entries' `entity/*` and `tech/*` tags,
- normalises each through the existing `normalize_tag` path (always treating
  `entity`/`tech` as reserved so variant spellings collapse),
- promotes canonical tags appearing on >= `threshold` entries to a single
  `entity` node keyed idempotently on `metadata.source_tag`,
- creates a `mentions` edge entry->node for every tagged entry via
  `INSERT OR IGNORE` on the `(from_id, to_id, relation_type)` unique index,
- returns `{entities_created, entities_reused, mentions_created}`.

Fully idempotent: a second run reuses the existing node and inserts 0 edges.
