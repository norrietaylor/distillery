## Implementation

Added `_create_structural_edges()` method to `GitHubSyncAdapter` in
`src/distillery/feeds/github_sync.py`. Integrated into both `sync()` and
`sync_batched()` via the same deferred-resolution pattern used for cross-ref
link relations.

Key behaviors:
- `citation` edges created for PRs with closing keywords referencing issues (R3.1)
- `depends_on` edges created for sub-issues with tracked-by/parent refs (R3.2)
- Missing target entries are skipped silently with a DEBUG log (R3.3)
- Edge writes are idempotent: `add_relation` returns the existing UUID when the
  `(from_id, to_id, relation_type)` triple already exists (R3.3)

Also updated `_process_issue_batch()` return type from a 3-tuple to a 4-tuple to
include `pending_structural` pairs alongside `pending_xrefs`.
