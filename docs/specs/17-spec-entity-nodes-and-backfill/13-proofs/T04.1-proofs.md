# T04.1 Proof Summary

**Task:** Resolve in-content URLs/#refs to external_id and materialize citation
edges in `reconcile_relations()`.

**Spec:** docs/specs/17-spec-entity-nodes-and-backfill/in-content-reference-resolution-in-reconcile.feature

## Implementation

- Added `backfill_relations_from_content_refs()` to
  `src/distillery/store/migrations.py`. It scans `entries.content` for absolute
  URLs and bare `#<number>` references, resolves them to entries via
  `metadata.external_id`, and inserts `citation` edges using the existing
  `INSERT OR IGNORE` pattern on the `(from_id, to_id, relation_type)` unique
  index. Self-edges, dangling targets, and ambiguous `#<number>` mappings are
  skipped.
- Wired the new pass into `DuckDBStore._sync_reconcile_relations()` inside the
  existing transaction. The returned dict now includes a distinct
  `content_ref_links` key alongside `metadata_links` and `wikilink_links`, and
  `total` sums all three.
