# T01.1 Proof Summary: Extend _handle_metrics with scope parameter

## Task

Add optional `scope` parameter (`"summary"` | `"full"` | `"search_quality"`, default `"full"`) to `_handle_metrics` in `analytics.py`. When `scope=summary`, return entry counts, db size, and embedding model info (former `distillery_status` payload). When `scope=search_quality`, return search totals and feedback rates (former `distillery_quality` payload). When `scope=full`, return the existing full metrics payload. Update the `distillery_metrics` tool registration in `server.py` to document and expose the new parameter.
