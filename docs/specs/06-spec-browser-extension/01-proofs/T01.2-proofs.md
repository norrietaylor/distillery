## Implementation Details

### Protocol Compliance
- JSON-RPC 2.0 envelope: `{"jsonrpc": "2.0", "id": N, "method": "...", "params": {...}}`
- Notifications omit `id` field per JSON-RPC 2.0 spec
- Headers: `Content-Type: application/json`, `Accept: text/event-stream`
- `Mcp-Session-Id` captured from initialize response, sent on subsequent requests
- `Authorization: Bearer {token}` included when auth token is set

### SSE Response Parsing
- Handles both `text/event-stream` and plain JSON responses
- Extracts JSON from `data:` lines, skips `[DONE]` sentinel
- Throws `MCPProtocolError` on JSON-RPC error responses

### Error Handling
- Network failures throw `MCPNetworkError` (caught by offline queue in T04.3)
- HTTP 401 throws `MCPAuthError` (caught by auth module in T03.1)
- HTTP 429 throws `MCPRateLimitError` with `retryAfter` seconds
- Other HTTP errors throw `MCPProtocolError` with status code
