Explanation:
The original code attempted to use chrome.runtime.sendMessage({action: 'getConnectionStatus'})
from within the background service worker. This doesn't work because sendMessage from a
service worker doesn't deliver to its own onMessage listener — the message is lost.

The fix directly accesses the in-scope connectionState variable (defined at line 65),
which is already maintained by the connect() and other lifecycle functions. This is
simpler, more efficient, and actually works as intended.
