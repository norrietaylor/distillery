# T04.1 Proof Summary: Feed Detection in Content Script

## Task
Extend `browser-extension/src/content.js` to detect RSS/Atom feeds and GitHub repos on the current page, and handle `feedsDetected` messages in the background service worker.

## Files Modified
- `browser-extension/src/content.js` — added `detectFeeds()`, `notifyFeedsDetected()`, DOMContentLoaded trigger, and `detectFeeds` message handler
- `browser-extension/src/background.js` — added `detectedFeeds` state variable, `feedsDetected` message handler (sets badge), `getDetectedFeeds` action handler, and `chrome.tabs.onUpdated` listener to reset feeds on navigation

## Implementation Summary

### content.js additions
- `detectFeeds()`: queries `link[rel="alternate"][type*="rss"]` and `link[rel="alternate"][type*="atom"]` elements, resolves relative URLs, classifies as `rss` or `atom`; also matches `github.com/{owner}/{repo}` pattern as `source_type: 'github'`
- `notifyFeedsDetected()`: calls `detectFeeds()` and sends `{ type: 'feedsDetected', feeds }` to background
- Triggers on `DOMContentLoaded` or immediately if DOM is already ready (`document_end`)
- Added `detectFeeds` message handler for on-demand popup requests

### background.js additions
- `detectedFeeds`: module-level array tracking latest detected feeds
- `feedsDetected` handler: updates `detectedFeeds`, sets indigo badge with feed count
- `getDetectedFeeds` action: returns `detectedFeeds` to popup (used by Watch tab in T04.2)
- `chrome.tabs.onUpdated` listener: clears `detectedFeeds` on navigation, restores queue badge
