The module implements GitHub OAuth 2.0 web application flow using
`chrome.identity.launchWebAuthFlow` and exports the full auth API:

- `startOAuthFlow(clientId, serverUrl)` — opens GitHub authorization via `chrome.identity.launchWebAuthFlow`, extracts authorization code from redirect, exchanges code for access token via MCP server's `/oauth/token` endpoint, fetches GitHub username from GitHub API, persists both to `chrome.storage.local`, returns `{token, username}`
- `getStoredAuth()` — retrieves `{token, username}` from `chrome.storage.local`
- `clearAuth()` — removes `authToken` and `authUsername` from storage (sign out)
- `getUsername()` — returns stored GitHub username or `null`
- `isAuthenticated()` — returns `true` if a non-null token is stored

Two custom error classes for downstream consumers:
- `AuthError` — general auth failures (exchange error, missing client ID, etc.)
- `AuthCancelledError` — user-cancelled or closed OAuth popup

## Implementation Details

### OAuth Flow
1. `chrome.identity.getRedirectURL('github')` provides the extension redirect URI
2. GitHub authorization URL built with `client_id`, `scope=user`, `redirect_uri`
3. `chrome.identity.launchWebAuthFlow({ url, interactive: true })` opens the popup
4. Authorization code extracted from the `?code=` query parameter of the callback URL
5. Code exchanged via `POST {serverUrl.replace(/\/mcp\/?$/, '')}/oauth/token`
6. GitHub username fetched from `https://api.github.com/user` using the token
7. Both persisted to `chrome.storage.local` as `authToken` and `authUsername`

### Token Endpoint Convention
The token endpoint is derived from the MCP server URL by stripping the `/mcp`
suffix and appending `/oauth/token`. For the production server:
`https://distillery-mcp.fly.dev/mcp` → `https://distillery-mcp.fly.dev/oauth/token`

### Error Handling
- Missing `clientId` → `AuthError` (fast-fail before network I/O)
- `launchWebAuthFlow` callback with `undefined` → `AuthCancelledError`
- `launchWebAuthFlow` callback with error-bearing `lastError` → `AuthCancelledError` or `AuthError`
- `?error=` in redirect URL → `AuthError` with GitHub's error message
- Token exchange HTTP error → `AuthError('Token exchange failed: ...')`
- Username fetch failure → silently returns `null` (non-blocking)

### Storage Keys
- `authToken` — GitHub access token (consistent with background.js usage)
- `authUsername` — GitHub login name (consistent with background.js usage)
