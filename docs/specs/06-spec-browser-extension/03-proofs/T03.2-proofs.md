## Auth Flow

1. User opens popup (remote, unauthenticated) → sees "Sign in with GitHub" button
2. User clicks → popup sends `startOAuth` to background
3. Background calls `startOAuthFlow(clientId, serverUrl)` from auth.js
4. OAuth completes → background applies token to mcpClient, reconnects, broadcasts updated state
5. Popup receives `connectionStatusChanged` → shows username + "Sign out" button

## Sign-out Flow

1. User clicks "Sign out" → popup sends `signOut` to background
2. Background calls `clearAuth()`, clears token on mcpClient, reconnects
3. Background broadcasts updated state (no username)
4. Popup shows "Sign in with GitHub" button again

## 401 Error Flow

1. MCP request returns 401 → existing `handleAuthError()` fires
2. Clears stored token, sets `connected: false`, broadcasts
3. Popup `updateAuthUI` detects `serverType: 'remote'` + `connected: false` → shows sign-in button
