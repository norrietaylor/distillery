## Changes Made

### src/distillery/store/duckdb.py

1. **Constructor**: Added `hybrid_search`, `rrf_k`, `recency_window_days`, `recency_min_weight` keyword-only parameters and `_fts_available` flag.
2. **`_setup_fts()`**: New method to install/load the FTS extension with graceful fallback.
3. **`_rebuild_fts_index()`**: New method to rebuild the FTS index via `PRAGMA create_fts_index` with `overwrite=1`. Called during `initialize()`, after `store()`, and after `update()` when content changes.
4. **`_bm25_search(query, limit)`**: New private method returning `list[tuple[str, int]]` (entry_id, rank) pairs from BM25 full-text search.
5. **`_recency_weight(created_at)`**: New method computing linear recency decay from 1.0 to `recency_min_weight` based on `recency_window_days`.
6. **`search()`**: Modified to fuse vector + BM25 via Reciprocal Rank Fusion (RRF) with recency decay when hybrid mode is active. Falls back to vector-only when FTS is unavailable or `hybrid_search=False`. BM25-only results are filtered through the same metadata filters as vector results.
7. **`_touch_accessed()`**: Extracted static helper for updating `accessed_at` timestamps.
8. **`_sync_initialize()`**: Calls `_setup_fts()` when `hybrid_search=True`, rebuilds FTS index after migrations, and logs INFO indicating whether hybrid or vector-only search is active.
