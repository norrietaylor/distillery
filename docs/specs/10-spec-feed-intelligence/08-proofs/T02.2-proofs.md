**Task**: T02.2 — Integrate topic tags into poll flow with vocabulary caching
**Completed**: 2026-04-06
**Model**: sonnet

## Implementation

Modified `src/distillery/feeds/poller.py`:

1. Updated `_item_to_entry_kwargs` to accept an optional `keyword_map: dict[str, str] | None` parameter. When provided, calls `derive_all_tags()` for Tier-1 + Tier-2 tags; when absent, falls back to `_derive_source_tags()` (source tags only).

2. Updated `poll()` to:
   - Call `store.get_tag_vocabulary()` once per poll cycle
   - Build keyword map via `build_keyword_map(vocabulary)`
   - Pass keyword map to `_poll_source()` via `keyword_map` keyword argument
   - Gracefully fall back to empty keyword map if vocabulary fetch fails

3. Updated `_poll_source()` to accept `keyword_map: dict[str, str] | None = None` and pass it through to `_item_to_entry_kwargs()`.

Added 5 new tests in `tests/test_feeds.py` class `TestPollCycleTopicTagIntegration`:
- `test_item_to_entry_kwargs_with_keyword_map_adds_topic_tags` — unit test for keyword_map parameter
- `test_item_to_entry_kwargs_without_keyword_map_only_source_tags` — unit test for None fallback
- `test_item_to_entry_kwargs_empty_keyword_map_only_source_tags` — unit test for empty map
- `test_poll_with_vocabulary_produces_topic_tagged_entries` — integration test for full poll cycle
- `test_poll_without_vocabulary_still_stores_source_tagged_entries` — resilience test
