## Implementation Details

### stores.ts Changes
- Added `TabType` export: `type TabType = "home" | "explore"`
- Added `activeTab` store: `writable<TabType>("home")`
- Default to "home" tab on app initialization

### NavBar.svelte Changes
- Import `activeTab` and `TabType` from stores
- Added tabs configuration with Home and Explore labels
- Tab buttons use onclick to call selectTab function
- Active tab indicated by border-bottom and color change
- ARIA attributes for accessibility (aria-label, aria-current)

### App.svelte Changes
- Import activeTab store and ExploreTab component
- Replace monolithic render with conditional blocks:
  - `$activeTab === "home"`: Render BriefingStats, RecentCorrections, ExpiringSoon, RadarFeed
  - `$activeTab === "explore"`: Render ExploreTab component
- Maintains error handling and connection states

### ExploreTab.svelte (New)
- Shell component with bridge prop interface
- Placeholder content: "Explore Tab - Advanced search coming soon"
- Consistent styling using Catppuccin theme variables
- Ready for future implementation of search/explore functionality
